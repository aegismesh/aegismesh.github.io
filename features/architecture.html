Stateless Edge Architecture

Modern service meshes often rely on bloated architectures requiring heavy sidecars (Envoy), centralized identity servers (Keycloak), and external state stores (Redis). AegisMesh strips away the bloat to give you raw performance and operational simplicity.

Java 25 & Virtual Threads

Built from the ground up for Project Loom. AegisMesh utilizes Virtual Threads to handle tens of thousands of concurrent I/O requests. By offloading heavy CPU tasks—like WebAssembly execution and ONNX vector embeddings—to dedicated bulkheaded thread pools, the gateway guarantees microsecond routing latency without thread starvation.

Embedded TCP Infinispan

External dependencies like Redis create licensing headaches and network hops. AegisMesh uses an embedded Infinispan cluster configured with TCPPING. When deployed to Kubernetes, gateway nodes discover each other statelessly via standard K8s DNS, forming a highly optimized, distributed rate-limiting cache natively inside the JVM.

Zero-Dependency Identity

We dropped heavy OIDC clients. AegisMesh operates as a stateless OAuth2 Resource Server. It resolves Ambient Kubernetes identities, validates M2M API keys cryptographically via SHA-256, and statically caches JWKS public keys to verify JWTs entirely in memory. This eliminates database lookups on the critical path.

Banking-Grade Security

Built in Zug, Switzerland, AegisMesh is engineered to meet the strictest FINMA and GDPR data sovereignty requirements. Security is not a plugin, it is the core routing foundation.

AST-Based Data Loss Prevention (DLP)

Traditional API gateways apply brute-force Regex over raw HTTP byte arrays. This causes Regex Denial of Service (ReDoS) CPU spikes and frequently corrupts embedded base64 binaries. AegisMesh parses JSON into a strict Abstract Syntax Tree (AST) using Jackson, traversing only textual values to redact sensitive PII (like Swiss IBANs or Credit Cards) without altering the structural integrity or binary payloads.

Hierarchical GitOps RBAC

Authorization is defined entirely via declarative YAML. AegisMesh features a two-tier policy engine: Cluster Master Policies enforce non-negotiable global constraints (like DDoS limits and inbound DLP), while Tenant Namespace Policies allow product teams to define granular endpoint roles. Any deviation from the GitOps source of truth is instantly blocked with a 403 Forbidden.

Ambient Identity & Zero-Trust

Never trust the network. AegisMesh resolves identities from multiple vectors simultaneously: Kubernetes Service Accounts (Ambient Mesh), mTLS client certificates, and cryptographically verified API Keys (via SHA-256 hash lookup). Spoofed internal headers are violently stripped. Every L7 routing decision is seamlessly upgraded to a short-lived, locally signed RSA-4096 JWT for secure downstream context.

WebAssembly (WASM) Extensibility

Standard API Gateways limit you to their built-in plugins or force you to write slow, out-of-process Lua/Python scripts. AegisMesh embraces the future of edge compute by allowing tenants to execute highly optimized WebAssembly (WASM) binaries directly in the request path.

Zero-JNI Execution via Chicory

Other gateways (like Envoy or Kong) use heavy C++ V8 engines to run Wasm, relying on dangerous Java Native Interface (JNI) bridges. A segmentation fault in a tenant's C++ plugin can crash the entire gateway. AegisMesh utilizes Chicory 1.x, a zero-dependency, pure-Java WebAssembly runtime. Wasm plugins run strictly within the JVM sandbox, guaranteeing 100% memory safety.

Zero-GC Primitive Performance

To ensure microsecond latency, AegisMesh bypasses traditional object-wrapper overhead. Utilizing modern Chicory APIs, we pass primitive 64-bit numbers (LongArrays) directly into the WebAssembly linear memory sandbox, executing complex mathematics and payload manipulations with absolute zero Garbage Collection (GC) overhead.

Bring Your Own Middleware (BYOM)

Your platform engineering teams are not restricted to writing Java plugins. Developers can write their custom L7 middleware in Rust, Go, Zig, or TypeScript, compile it to a standard .wasm binary, and push it to your GitOps repository. AegisMesh dynamically mounts the binary and executes it on the fly.

AI-Ready Edge Proxy

Integrating Large Language Models (LLMs) into enterprise infrastructure introduces extreme latency, massive API costs, and severe prompt injection risks. AegisMesh intercepts outbound AI traffic to sanitize, cache, and govern it natively at the edge.

Local ONNX Semantic Caching

Exact-string caching is useless for AI. "What is GitOps?" and "Can you explain GitOps?" mean the exact same thing but have different strings. AegisMesh uses LangChain4j 1.x to run a quantized ONNX embedding model (all-minilm-l6-v2) completely locally inside the JVM. It converts incoming prompts into dense vectors in microseconds, bypassing external embedding APIs entirely.

VectorDB Tenant Isolation

Semantic caches are dangerous in multi-tenant environments if not strictly segregated. Vectors are queried against an attached Milvus database. AegisMesh injects strict tenantId and Time-To-Live (TTL) metadata filters into the VectorDB queries, guaranteeing that semantic hits never leak data across organizational boundaries or serve outdated intelligence.

Token Quotas (TPM SLA)

Rate limiting AI by "requests per minute" is obsolete. AegisMesh intercepts the payload, runs it through a CL100K_BASE tokenizer, and counts the exact tokens. It enforces strict Tokens-Per-Minute (TPM) SLAs on a per-user and per-tenant basis via our embedded Infinispan cluster, dropping traffic with a 429 if a tenant exceeds their LLM budget.
